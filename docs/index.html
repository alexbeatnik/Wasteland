WASTELAND TERMINAL v0.4

A highly secure, offline-first LLM chat application with a retro-futuristic TUI-like graphical interface. Built in pure C11 with a vintage PC-inspired amber-on-black CRT terminal aesthetic. All inference happens locally via llama.cpp. Once a model is loaded, Linux seccomp physically prevents the process from opening any new IP socket for the rest of its lifetime.

--- CORE PROTOCOLS ---

Offline-first All inference happens locally via llama.cpp. No cloud, no telemetry.
Hard Network Lockdown Linux seccomp kills the process if it opens a new AF_INET / AF_INET6 / AF_PACKET socket after load.
Retro UI Amber (#FFB000) monochrome CRT terminal aesthetic via Nuklear; threaded 60 FPS UI with background inference.
Async Model Load UI stays responsive during multi-second GGUF load; cancel in-flight downloads.
Smart Reasoning Automatic <think> block detection with dimmed rendering; excluded from copy-to-clipboard.
Context Compaction Drop oldest turns, save compacted history to disk, and mirror into inference without reload.
Agent Mode (ReAct) Built-in tool-using ReAct agent loop with sandboxed file system access (read_file, list_dir, write_file, apply_edit).
Auto-Generated Chat Titles First message names the chat; the model then generates a contextual 3–5 word title and the file is renamed automatically.
One-Click Auto-Update Background GitHub check on startup. Download + replace + restart with a single button press — UAC / pkexec handled automatically.
Multi-Turn Memory Create, load, and switch between named chat sessions. Persisted locally with RC4 obfuscation.

--- SECURITY MODEL ---

  1. App boots in offline-capable state with the network reachable so you can download models.
  2. Nothing is loaded automatically. You pick a model via the UI.
  3. When you click [ LOAD ] and the load succeeds, lockdown_network() runs:
    • Linux: seccomp-bpf filter kills the process on any new socket(AF_INET, ...).
    • macOS / Windows: no-op (platform limitation).
  4. seccomp filters cannot be removed for the lifetime of the process. Unloading a model does not lift the lockdown — restart to download more models.
  5. The download path is gated by state->network_lockdown in the UI, so the [ DOWNLOAD ] button hides the moment the lockdown is active.
